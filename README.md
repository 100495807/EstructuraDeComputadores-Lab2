# EstructuraDeComputadores-Lab2

Practica de Estructura de Computadores centrada en microprogramacion y definicion de instrucciones. El repositorio contiene ejercicios donde se describen instrucciones mediante campos de control, registros, memoria y saltos de microcodigo.

## Objetivo

Comprender como se implementan instrucciones de arquitectura a nivel de ruta de datos, senales de control y secuencias de microoperaciones.

## Contenido

| Archivo | Descripcion |
| --- | --- |
| `exercise_1` | Definicion de instrucciones y microoperaciones para carga, almacenamiento, operaciones complejas y saltos. |

## Conceptos Trabajados

- Ciclo de instruccion y fetch.
- Registros internos como MAR, MBR, PC y registros temporales.
- Lectura y escritura de memoria.
- Senales de control de la ruta de datos.
- Instrucciones de carga y almacenamiento.
- Operaciones aritmeticas y comparaciones.
- Saltos condicionales en microcodigo.

## Como Usarlo

El archivo esta pensado para el entorno de simulacion de la asignatura. El flujo habitual es:

1. Abrir `exercise_1` en el simulador.
2. Cargar la arquitectura o definicion de instrucciones.
3. Ejecutar las instrucciones y observar la evolucion de registros y memoria.
4. Validar que cada secuencia termina correctamente en el ciclo de fetch.

## Aprendizajes

- Traducir una instruccion de alto nivel a microoperaciones.
- Razonar sobre la ruta de datos y las senales necesarias en cada ciclo.
- Coordinar accesos a memoria, ALU y escritura en registros.
- Depurar errores en instrucciones observando estados intermedios.
